EVALUATION PLAN

Evaluation Plan pada MySQL

Pada MySQL cara menampilkan evaluation plan menggunakan syntax EXPLAIN. Untuk Database pada gambar di samping saya menggunakan perintah :

EXPLAIN SELECT customers.id, customers.nama, details.pesanan, details.jmlbarang FROM customers INNER JOIN details ON customers.id = details.id

Hasil eksekusi perintah

id

identifier SELECT. Nomor urut dari SELECT yang ada dalam query. Kedua tabel memiliki id yang sama yaitu 1.

type

kolom ini menjelaskan bagaimana tabel di join atau bergabung. Pada query ini terdapat ALL untuk tabel customers dimana dilakukan scanning lengkap oleh query pada suatu tabel dan eq_ref untuk tabel details dimana satu baris dibaca dari tabel ini untuk setiap kombinasi baris-baris dari tabel sebelumnya.

key_len

menunjukkan panjang dari key yang digunakan oleh MySQL. Nilai dari key_len menunjukkan berapa banyak bagian dari multiple-part key yang digunakan MySQL. Jika kolom key menunjukkan NULL, maka key_len nya juga akan NULL karena tidak ada panjang key jika key nya tidak ada. Untuk tabel details key_len nya 4.

select_type

tipe dari SELECT. Pada query ini yang digunakan adalah SIMPLE SELECT dimana tidak menggunakan UNION atau subqueries.

possible_keys

mengindikasi index dimana MySQL dapat memilih untuk menemukan baris/rows pada tabel. Untuk NULL berarti tidak ada indeks yang relevan, bisa juga menggunakan WHERE untuk mengecek apakah merujuk pada suatu kolom yang mungkin bisa cocok untuk indexingnya. Sedangkan PRIMARY berarti index relevannya adalah primary key.

ref

menunjukkan kolom atau konstanta mana yang dibandingkan dengan index yang disebut pada kolom key untuk memilih baris dari tabel. Sama seperti key_len, apabila nilai key nya NULL maka nilai ref nya juga NULL, yang berarti tidak ada kolom atau konstanta yang bisa dibandingkan dengan index yang disebut pada kolom key karena memang bernilai NULL. Sedangkan untuk tabel details ref nya menunjuk ke customers.id seperti yang sudah ada pada kolom type yaitu eq_ref yang berarti membaca satu baris dari tabel details untuk setiap kombinasi baris-baris dari tabel customers.

table

nama dari tabel untuk baris output.

key

menunjukkan key atau index yang dipilih untuk digunakan oleh MySQL. Jika key yang dipilih MySQL merupakan salah satu index dari possible keys, maka index itu terdaftar sebagai key value. Untuk key NULL berarti MySQL tidak menemukan index yang dapat digunakan untuk mengeksekusi query dengan lebih efisien.

rows

menunjukkan baris yang akan diperiksa oleh MySQL untuk mengeksekusi query. Untuk tabel customers, row nya ada 5 dan untuk tabel details ada 1. Semakin tinggi nilai row nya maka semakin baik tingkat optimasi mengenai index. Namun dapat juga memperlambat kecepatan dan menurunkan efisiensi kerja.

Extra

informasi tambahan bagaimana MySQL menyelesaikan query. Kedua tabel yaitu customers dan details tidak menggunakan Extra.
