QUERY

Mengukur biaya sebuah Query

Ada beberapa kemungkinan evaluation plan untuk sebuah query, dan itu penting untuk bisa membandingkan alternatif-alternatif plan dalam hal perkiraan biayanya, sehingga dapat memilih plan terbaik. untuk melakukan itu, kita harus memperkirakan biaya dari operasi-operasi individual dan membandingkannya untuk bisa mengetahui biaya dari evaluation plan sebuah query.

Biaya sebuah query dapat diukur dari banyaknya sumber daya sistem yang digunakan, misalnya : pengaksesan disk, waktu CPU mengeksekusi query, dan waktu komunikasi pada sistem database terdistribusi. biaya query juga bisa dihitung dengan cara cross join, yaitu dengan cara mengalikan jumlah row di masing-masing tabel. Pada query, umumnya terdapat berbagai metode untuk menghitung jawabannya. query optimizer berperan untuk merubah query ketika dimasukkan oleh user menjadi query ekuivalen yang dapat dihitung dengan lebih efisien.

Perbedaan biaya (dalam hal evaluation time) antara strategi yang baik dan buruk biasanya substansial dan mungkin bisa beberapa kali lipat.oleh karena itu, ada baiknya bagi sebuah sistem untuk menghabiskan banyak waktu pada pemilihan strategi yang baik untuk memproses query, meskipun query tersebut hanya dieksekusi sekali.

Evaluasi Single & Multiple Expression

Cara untuk mengevaluasi sebuah ekspresi adalah dengan mengevaluasi satu operasi pada satu waktu, dengan urutan yang sesuai. hasil dari setiap evaluasi di materialisasikan dalam relasi sementara untuk penggunaan selanjutnya. kerugian dari pendekatan ini adalah kebutuhan untuk membangun relasi sementara, yang harus di tulis ke disk kecuali ukurannya kecil. pendekatan alternatif adalah dengan mengevaluasi beberapa operasi secara bersamaan dalam pipeline, dengan hasil dari satu operasi diteruskan ke operasi selanjutnya, tanpa perlu menyimpan relasi sementara.

Materialization

Evaluasi Materialized merupakan evaluasi dimana hasil dari setiap operasi intermediate dibuat (materialized) dan kemudian digunakan untuk evaluasi operasi di level selanjutnya. cara paling mudah untuk memahami secara intuitif bagaimana mengevaluasi sebuah ekspresi adalah dengan melihat representasi bergambar dari ekspresi dalam sebuah operator tree. jika kita menerapkan pendekatan materialization, kita mulai dari operasi level terendah dalam ekspresi.

Pipelining

Pipeline Evaluation merupakan evaluasi dimana kita dapat meningkatkan efisiensi evaluasi query dengan cara mengurangi jumlah file temporary/sementara yang dibuat. pengurangan ini didapatkan dengan cara menggabungkan beberapa operasi relasional ke dalam pipeline dari operasi, di mana hasil dari satu operasi diteruskan ke operasi selanjutnya di dalam pipeline.

Evaluasi Single & Multiple Expression

Secara default, ketika ekspresi dan item data diteruskan ke operator evaluate, sebuah query dinamis dikeluarkan untuk mengevaluasi ekspresi item data. Sehingga untuk satu set multiple ekpresi, setiap ekspresi membutuhkan satu query dinamik. pendekatan dalam pengujian setiap ekspresi sebuah item data merupakan solusi waktu linear dan tidak dapat diskalakan untuk set ekspresi yang besar yang dievaluasi terhadap item data dengan ukuran yang tinggi. untuk tujuan ini, sebuah indeks opsional dapat didefinisikan pada kolom yang menyimpan file ekspresi untuk mengevaluasinya dengan efisien.
